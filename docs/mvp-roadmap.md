# MVP 路线图

## 1. 目标

在 `12` 周内完成首版产品验证，最小可用范围聚焦在“注册、认证、建档、推荐、首聊、会员”六个关键节点。

## 2. 交付范围

### 2.1 App 端

- 手机号注册登录
- 实名认证
- 真人认证
- 轻人格与兴趣建档
- 推荐流
- 翻牌匹配
- 私聊
- 举报/拉黑
- 会员中心

### 2.2 小程序端

- 品牌介绍
- 轻测试
- 匹配报告预览
- App 下载引导

### 2.3 后台

- 用户审核
- 举报处理
- 标签配置
- 会员套餐配置
- 基础数据看板

## 3. 12 周节奏

### 第 1-2 周：需求定稿与原型

- 冻结一期范围，明确不做项
- 产出信息架构、低保真原型、交互流程
- 确定实名、真人认证、IM、审核服务商

验收：

- 所有核心页面有明确原型
- 第三方服务选型完成

### 第 3-4 周：视觉设计与技术底座

- 完成品牌视觉、核心页面 UI
- 搭建账号、用户资料、标签、会员、举报基础模型
- 接入短信、支付、基础埋点

验收：

- 核心设计稿完成
- 服务端基础表结构和接口文档完成

### 第 5-8 周：核心功能开发

- 开发注册、认证、建档、推荐、匹配、私聊
- 开发小程序轻测试和引流页
- 开发后台审核和会员配置

验收：

- 走通新用户完整主流程
- 后台可完成基本审核和封禁

### 第 9-10 周：联调与灰度

- 联调实名认证、真人认证、支付、IM
- 完成埋点验收和风控策略首版
- 开始小规模种子用户灰度

验收：

- 主流程无阻塞级 bug
- 可观察注册、认证、匹配、首聊、付费漏斗

### 第 11-12 周：优化与上线准备

- 根据灰度数据调优推荐和建档体验
- 修复高频问题
- 准备活动拉新素材、FAQ、审核话术

验收：

- 可进入单城正式验证
- 运营与审核具备上线值守能力

## 4. 角色分工

- 产品经理：范围控制、原型、PRD、数据口径
- 设计师：品牌与核心流程视觉
- 客户端工程师：App 主流程实现
- 后端工程师：账号、推荐、会员、审核后台
- 测试：流程回归、兼容与风控场景
- 运营：冷启动社群、活动和审核策略

## 5. 首版不做项

- 重内容社区
- 复杂直播房
- 大规模动态推荐算法
- 多城市同时铺开
- 复杂商城体系

## 6. 核心验收指标

- 认证完成率 `> 60%`
- 首日首聊率 `> 25%`
- 匹配到有效对话转化率 `> 15%`
- 举报率可控且能闭环处理
- 首批付费率具备可验证空间

## 7. 上线后 30 天重点观察

- 哪类兴趣标签组合最容易转化为有效对话
- 认证门槛对注册流失的真实影响
- 会员权益中最能驱动付费的功能点
- 单城用户密度是否足以支撑持续匹配体验
